淺談 Webpack (1) 什麼是 Webpack ?

2018-10-20 20:36:02
FrontEnd

寫在最前面

這是一個系列的文章，會想寫這系列是因為 Webpack 在現在前端開發是一個滿重要的存在，雖然他跟寫 Code 沒有太直接的關係，就算你不會它也可以把 JavaScript 寫得很好，但如果沒有它你可能會在開發上遇到一些麻煩，一直以來我都是依賴 Vue CLI 來幫我建立開發環境導致我對 Webpack 一直是一知半解，所以希望透過這個系列的學習能夠真正的搞懂 Webpack 究竟是什麼東西，也希望能夠幫助剛開始學習 Webpack 的人，如果文章有哪裡寫錯請多多包涵並指教，謝謝。

什麼是 Webpack ?

webpack 是一個 JavaScript 的打包工具，負責將整個網頁所需要用到的 JavaScript 檔打包成一個。

為什麼使用 Webpack ?

在以前剛開始學習寫 JavaScript 的時候常常一個網頁只有一支 JavaScript 檔案，但裡面可能有幾百行幾千行程式碼，這樣的寫法在日後維護起來實在是很麻煩。光是要找到需要修改的程式碼就必須花上好一段時間，更別說找到以後還要擔心改了這個會不會壞了另一個。

所以為了解決這個問題，大家也慢慢的把 JavaScript 模組化了，但這樣還是有個缺點，就是 index.html 需要引入的 JavaScript 檔案變多了，檔案一多對頁面載入的效能造成引響，而且如果 A 檔案會用到 B 檔案裡面的程式碼，那 A 檔案就必須要在 B 檔案之後載入，不然可能會有錯誤。

所以為了解決這個問題，大家也慢慢的把 JavaScript 模組化了，但這樣還是有個缺點，就是 index.html 需要引入的 JavaScript 檔案變多了，檔案一多對頁面載入的效能造成引響，而且如果 A 檔案會用到 B 檔案裡面的程式碼，那 A 檔案就必須要在 B 檔案之後載入，不然可能會有錯誤。

Webpack 就是為了解決這些問題而出現的。Webpack 會把你所有的 JavaScript 檔案打包成一個，更重要的是它會確保所有 JavaScript 載入的順序正確，也就是你再也不用為互相依賴的檔案煩惱了！
