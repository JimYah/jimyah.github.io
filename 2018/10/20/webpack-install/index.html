基本設定

接下來我們在這個目錄下新增以下幾個檔案

src/sum.js

sum.js 是一個可以接受兩個數值當參數並回傳總和的 Function ，這裡有用到 ES6 的 export default 為的是能讓 sum.js 可以在其它地方被 import。

src/index.js

接著我們在 index.html 裡面 import sum.js，並用 console.log 來 Print 出結果。

上面的步驟完成後我們就可以來對 Webpack 做設定，首先我們必須建立一個叫做 webpack.config.js 的檔案，之後當 Webpack 啟動以後會以這個檔案作為基準。

首先我們會先載入 Path 這個 node module ，它能提供一些有用的方法。接著你會看到 Webpack 設定檔其實只是一個物件，裡面包含兩個屬性 (entry & output)。

  • entry: 要求字串，載入點的相對路徑

我們先來說說 entry 是什麼東西，在 Webpack 裡面 entry (通常我們會把 Entry 命名為 index.js) 是整個程式的進入點，簡單來說 Ｗebpack 在處理你的 JavaScrtip 檔案的時候會先去抓 Entry 並且看它 Import 了哪些檔案，一直到把所有 Import 進來的檔案成功連結在一起，也因為這樣 entry 只會有許多的 import 不會有任何的 export。

  • output: 要求物件
    • filename: 要求字串，最後輸出的檔名
    • path: 要求路徑，最後輸出的絕對路徑

接著我們來說說 output，output 它是一個物件接受兩個 Property，第一個是最後輸出的檔名(通常我們會把它叫做 bundle.js)，再來是最後輸出的路徑，這裡官方文件用了 path.resolve 這個方法來解析路徑，而 __dirname 是 Node.js 裡面的一個方法它可以取得檔案當前的資料夾名稱，並與後面的參數結合在一起成為一個路徑，所以我們會取得一個叫做 ~/webpack-demo/dist 的路徑，而最後的輸出檔會放在 dist 資料夾裡面。

我們已經把 Webpack 基本的設定都用好了，接下來我們只要 Run 它就好了

下完指令以後你應該會看到 Webpack 一些編譯的訊息，如果成功你會看到你的專案目錄底下多了一個 dist 資料夾，裡面有會有一個 bundle.js。

另外我們還可以透過增加 npm script 的方法來讓 Webpack 跑起來

package.json

接著我們先把剛剛 build 出來的 dist 資料夾刪除，然後再下一次 npm run build，如果沒有錯的話會跟剛剛一樣多一個 dist 資料夾。

最後為了驗證我們是不是真的成功我們必須建立一個 index.html 必且引入 bundle.js

接著打開這個檔案看 console.log 如果有看到 20 那代表我們成功囉！
