淺談 Webpack (4) Babel Loader

2018-10-20 20:59:04
FrontEnd

為什麼使用 Babel ？

相信大家在開發網頁的時候都已經開始用 ES6 的語法了，但是基於一些比較舊版本的瀏覽器不支援 ES6 的語法，這個時候我們就需要 Babel 來幫我們解決這件事，Bable 可以幫我們把 ES6 的語法編譯成讓所有瀏覽器都支援得 ES5。

安裝

這篇文章主要會是在講 babel-laoder v8 + Babel v7，如果是要看之前的版本可以去官方的 GitHub ⇒ 傳送門

  1. @babel/core - 整個 babel 的核心，負責編譯 JavaScript。
  2. babel-loader - 負責跟 Webpack 做溝通
  3. @babel/preset-env - 像是一本規則書，babel 會將 preset-env 作為規則去編譯 ES6，另外我們也可以自己設定我們想要向下兼容到什麼樣的版本。在預設情況下 @babel/preset-env 會去抓 .browserslistrc 這個設定檔，除非，除非我們在 options 裡面有裡面有另外設定 targets 選項。

設定

首先我們現在專案目錄下新增 .browserslistrc

在這份檔案裡面我們就可以依照我們專案的需求去設定需要讓 ES6 的語法兼容到什麼程度。

到這裡我們在執行一次 npm run build，這個時候去看編譯出來的 bnudle.js 你會發現你的 JavaScript 語法都變成 ES5 的語法了!
