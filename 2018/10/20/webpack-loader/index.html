淺談 Webpack (3) 什麼是 Loader ?

2018-10-20 20:51:57
FrontEnd

在 webpack 的世界裡，webpack 只看得懂 JavaScript 檔案，所以它也只能對 JavaScript 做處理，而 loaders 可以讓 webpack 去看懂其它不同的檔案並處理它們，讓這些檔案變成可以載入在你的應用程式的有效”模組“。

基本上我們會給 loaders 兩個屬性：

  1. test: 指定哪一個檔案需要使用 loader 去處理。
  2. use: 指定哪個你要使用的 loader。

看起來會長這樣 webpack.config.js

在加入 loader 之前我們需要先在設定檔裡加一個屬性叫 module 它是一個物件，裡面會有一個叫 rules 的屬性它是一個陣列，而在 rules 裡面我們就可以放各式各樣不同的 loader 在裡面。
之後 loader 會告訴 webpack 說，Hey webpack 當你遇到 .txt 的檔案的時候先幫我用 raw-loader 處理然後再丟進 bundle.js。

CSS Loader & Style Loader

CSS loader 可以讓我們用 import 的方式在 JS 檔裡面載入 CSS 檔案，而 style loader 能幫我們把載入的 CSS 放到 header 裡面並用 style tag 包住。

接下來我們就先來安裝這兩個 loader 吧！

style.css

再來我們執行 webpack 並打開 index.html

這個時候你應該會看到 Hello Webpack 並沒有變成粉紅色的，這是因為前面我們有說過 CSS loader 只是讓我們可以在 JS 裡面 import CSS 的檔案，但它並不會幫我們把 CSS 放到網頁上，所以我們還需要 style loader 的幫忙。

引入 style loader

這裡有一個重點 webpack 在執行 loader 的時候是由下往上，以上面為例 webpack 會先執行 css loader 把 css 檔轉換成 JS 可編譯的檔案，然後再執行 style loader 把 css 放進 header 的 style tag。

接著我們在執行一次 webpack，如果看到 Hello Webpack 變成粉紅色的就代表成功了，這時你可以打開開發者工具，你會看到 webpack 幫我們把 css 放到 head 裡面的 style tag 了。
